# Handson 1: First Web API Using ASP.NET Core

## Scenario

Modern applications often expose functionality through Web APIs so that different clients such as web applications, mobile applications, and other services can communicate using HTTP requests and responses.

ASP.NET Core provides a powerful framework for building RESTful Web APIs using controllers, action methods, routing, and HTTP verbs.

In this handson, a simple ASP.NET Core Web API was created and tested by executing a GET request and verifying the returned response.

---

## Objective

The objective of this handson is to:

- Create an ASP.NET Core Web API project.
- Understand the structure of a Web API application.
- Create a controller with action methods.
- Implement HTTP action verbs such as GET, POST, PUT, and DELETE.
- Execute the Web API application.
- Verify that the GET action method returns the expected response.

---

## Project Structure

A new ASP.NET Core Web API project named **FirstWebApi** was created using Visual Studio Community.

The project contains the default files generated by the ASP.NET Core Web API template along with a custom controller named **ValuesController**.

## Implementation Steps

### Step 1: Create the ASP.NET Core Web API Project

A new ASP.NET Core Web API project was created using Visual Studio Community.

The project template automatically generated the required configuration files, controller support, and application startup configuration.

---

### Step 2: Create ValuesController

A new controller named **ValuesController** was created inside the **Controllers** folder.

The controller inherits from `ControllerBase` and is configured using routing attributes.

```csharp
[Route("api/[controller]")]
[ApiController]
public class ValuesController : ControllerBase
{
}
```

### Explanation

- `[Route("api/[controller]")]` defines the route pattern for the controller.
- `[ApiController]` identifies the class as a Web API controller.
- The controller becomes accessible using the route:

```
/api/values
```

---

### Step 3: Implement HTTP Action Methods

The following action methods were implemented:

```csharp
[HttpGet]
public IActionResult Get()
{
    return Ok("GET Request Executed Successfully");
}

[HttpPost]
public IActionResult Post()
{
    return Ok("POST Request Executed Successfully");
}

[HttpPut]
public IActionResult Put()
{
    return Ok("PUT Request Executed Successfully");
}

[HttpDelete]
public IActionResult Delete()
{
    return Ok("DELETE Request Executed Successfully");
}
```

### Explanation

#### GET

The GET action method is used to retrieve data from the server.

```csharp
[HttpGet]
```

When a request is sent to:

```
/api/values
```

the API returns:

```
GET Request Executed Successfully
```

#### POST

The POST action method is used to create new resources.

```csharp
[HttpPost]
```

#### PUT

The PUT action method is used to update existing resources.

```csharp
[HttpPut]
```

#### DELETE

The DELETE action method is used to remove resources.

```csharp
[HttpDelete]
```

---

### Step 4: Execute the Application

The application was executed using Visual Studio.

ASP.NET Core started the Kestrel web server and hosted the Web API locally.

The application started successfully and listened for incoming HTTP requests.

---

### Step 5: Verify the GET Endpoint

The GET endpoint was tested by navigating to the following URL:

```
http://localhost:5171/api/values
```

> *(Use the port number generated on your system.)*

The API returned the expected response:

```
GET Request Executed Successfully
```

This confirmed that the controller, routing configuration, and GET action method were working correctly.

## Analysis

### ASP.NET Core Web API

ASP.NET Core Web API provides a framework for building RESTful services that communicate using HTTP.

A Web API receives HTTP requests from clients and returns HTTP responses containing data or status information.

### HTTP Request and HTTP Response

A client sends an HTTP request to the server.

The server processes the request and returns an HTTP response.

For this handson:

**Request**

```
GET /api/values
```

**Response**

```
GET Request Executed Successfully
```

### HTTP Action Verbs

HTTP verbs represent different operations that can be performed on resources.

| HTTP Verb | Purpose |
|-----------|---------|
| GET | Retrieve data |
| POST | Create data |
| PUT | Update data |
| DELETE | Remove data |

These verbs are implemented in ASP.NET Core using attributes such as:

```csharp
[HttpGet]
[HttpPost]
[HttpPut]
[HttpDelete]
```

### IActionResult and HTTP Status Codes

Each action method returns an `IActionResult`.

The `Ok()` method returns a successful HTTP response with status code:

```
200 OK
```

This indicates that the request was processed successfully.
